# Hola Amigo

## Supabase Setup For Quiz Words

Quiz words are now loaded from Supabase (table: `questions`) instead of local fake data.

### 1) Required environment variables

Create a `.env.local` file:

```bash
VITE_SUPABASE_URL=your_supabase_project_url
VITE_SUPABASE_PUBLISHABLE_KEY=your_supabase_anon_key
```

### 2) Create the `questions` table

Run this SQL in Supabase SQL Editor:

```sql
create table if not exists public.questions (
	id bigint generated by default as identity primary key,
	level text not null,
	prompt text not null,
	answers jsonb not null,
	correct integer not null,
	created_at timestamptz not null default now(),
	constraint questions_answers_array check (jsonb_typeof(answers) = 'array')
);

create index if not exists questions_level_idx on public.questions (level);
```

### 3) Add read policy (RLS)

If RLS is enabled on `questions`, allow authenticated users to read:

```sql
alter table public.questions enable row level security;

create policy if not exists "authenticated users can read questions"
on public.questions
for select
to authenticated
using (true);
```

### 4) Insert sample rows

```sql
insert into public.questions (level, prompt, answers, correct)
values
	('A1.2', 'Perdón', '["Excuse me / Sorry", "Thank you", "Goodbye", "Fine"]'::jsonb, 0),
	('A1.2', 'Me llamo Ana', '["My name is Ana", "I have Ana", "I''m from Ana", "You are Ana"]'::jsonb, 0),
	('A1.2', '¿Cómo estás?', '["What time is it?", "How are you?", "Where are you from?", "How old are you?"]'::jsonb, 1);
```

Important: each level needs at least 5 rows, because each round pulls 5 questions.
