# Hola Amigo

## Supabase Setup For Quiz Words

Quiz words are now loaded from Supabase (table: `questions`) instead of local fake data.

### 1) Required environment variables

Create a `.env.local` file:

```bash
VITE_SUPABASE_URL=your_supabase_project_url
VITE_SUPABASE_PUBLISHABLE_KEY=your_supabase_anon_key
```

### 2) Create the `questions` table

Run this SQL in Supabase SQL Editor:

```sql
create table if not exists public.questions (
	id bigint generated by default as identity primary key,
	difficulty_level text not null,
	spanish_word text not null,
	english_word text not null,
	created_at timestamptz not null default now()
);

create index if not exists questions_difficulty_level_idx on public.questions (difficulty_level);
```

### 3) Add read policy (RLS)

If RLS is enabled on `questions`, allow authenticated users to read:

```sql
alter table public.questions enable row level security;

do $$
begin
	if not exists (
		select 1
		from pg_policies
		where schemaname = 'public'
			and tablename = 'questions'
			and policyname = 'authenticated users can read questions'
	) then
		create policy "authenticated users can read questions"
		on public.questions
		for select
		to authenticated
		using (true);
	end if;
end
$$;
```

### 4) Insert sample rows

```sql
insert into public.questions (difficulty_level, spanish_word, english_word)
values
	('A1.2', 'Perdón', 'Excuse me / Sorry'),
	('A1.2', 'Me llamo Ana', 'My name is Ana'),
	('A1.2', '¿Cómo estás?', 'How are you?');
```

Important: each level needs enough distinct English words to build multiple choice options. A safe minimum is 5+ rows per level.
